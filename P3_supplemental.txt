QS1.1: 
We implemented computeActionFromValues by first initiliazing the best action and max q value 
to the minimum possible values, then we loop through all of the possible actions and resulting states.
We then get the q value from the possible future states and compare them to the previously initialized
best state and q value, updating them only if the new action has a higher q value. After looping through
all of the actions, the result is guaranteed to be the best action with the highest q value.

QS1.2:
ComputeQValueFromValues initializes the q value to 0, and then loops and updates the q value.
The loop goes through all the possible next states given an action and adds their value multiplied 
by their probability to the current state.

Q5#######################

QS5.1:
getQValue returns the Q-value for a (state, action) pair from a Counter, defaulting to 0.0 for unseen pairs. 
This means unseen actions can be optimal when all seen actions have negative Q-values.
computeValueFromQValues returns max_a Q(s,a) over legal actions, or 0.0 for terminal states.
computeActionFromQValues collects all actions tied for the highest Q-value and picks one randomly with random.choice() to avoid always favoring the same action.
update applies the Q-learning formula: Q(s,a) <- Q(s,a) + α[r + γ·max_a' Q(s',a') − Q(s,a)]. Each observed transition nudges the Q-value toward the sample estimate. 
This is off-policy, the max over next-state actions means we learn the optimal policy regardless of what action we actually take.
Unlike value iteration, Q-learning needs no transition model. It learns directly from (s, a, r, s') experience tuples.

Q6#######################
QS6.1:
With epsilon=0.1, the agent explores randomly only 10% of the time and exploits its learned Q-values 90% of the time. It quickly converges to the optimal policy and consistently
follows the best path, with only occasional random deviations. Returns are high and stable after the initial learning phase.

With epsilon=0.9, the agent acts randomly 90% of the time and only follows its best Q-values 10% of the time. It explores extensively
but rarely exploits what it has learned, so it wanders erratically and frequently stumbles into the -1 terminal or takes long detours. Average
returns are much lower. The Q-values may still converge toward optimal given enough episodes, but the agent's actual behavior
remains poor because it almost never follows its own policy.

This matches expectations: low epsilon favors exploitation, high epsilon favors exploration.
